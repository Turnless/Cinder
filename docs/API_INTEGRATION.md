# Cinder — API Integration Guide

This document details how Cinder integrates with the **SoSoValue API** (Data & Intelligence Layer) and the **SoDEX API** (Trading & Execution Layer).

---

## 🔷 1. SoSoValue API Integration

The SoSoValue API acts as the data foundation for Cinder's Wire Generator and Narrative Intelligence Engine.

### Authentication & Headers

All requests to the SoSoValue API must include the API key in the `Authorization` header:

```http
GET /v1/etf/btc/flow HTTP/1.1
Host: api.sosovalue.xyz
Authorization: Bearer YOUR_SOSOVALUE_API_KEY
Accept: application/json
```

---

### Core Endpoints Used

#### 1. Spot ETF Daily Net Flows
Retrieves the daily net flows and total assets for BTC or ETH spot ETFs.

- **Endpoint:** `GET /v1/etf/{asset}/flow`
- **Parameters:**
  - `asset` (path): `btc` or `eth`
  - `days` (query, optional): Number of historical days (default: `7`)

- **Example Response:**

## 🟠 2. SoDEX API Integration

SoDEX executes the trades generated by the Narrative Engine. Because SoDEX is a decentralized exchange running on ValueChain L1, it requires EIP-712 signatures for all state-changing write operations.

### Base URLs

- **Testnet Spot REST:** `https://testnet-gw.sodex.dev/api/v1/spot`
- **Mainnet Spot REST:** `https://mainnet-gw.sodex.dev/api/v1/spot`

---

### EIP-712 Authentication Protocol

To authenticate private REST requests:
1. Generate an **API Key** via the SoDEX UI. Note the Key Name (e.g. `Cinder-Key-1`).
2. Construct the EIP-712 Typed Data payload.
3. Sign the typed data using your Master EVM private key.
4. Pass the signature, key name, and nonce in the headers.

#### Required Headers
- `X-API-Key`: The human-readable name of the registered key.
- `X-API-Nonce`: Current epoch timestamp in milliseconds (e.g., `1783778400000`).
- `X-API-Sign`: The hex signature of the EIP-712 payload prefixed with `0x01`.

---

### Node.js Signing Implementation

Here is how Cinder implements EIP-712 signature generation in `src/lib/sodex.js`:

```javascript
import { ethers } from 'ethers';

// Domain depends on which market you're trading — not a fixed "SoDEX" domain
function getDomain(market, isMainnet) {
  return {
    name: market,                     // 'spot' or 'futures'
    version: '1',
    chainId: isMainnet ? 286623 : 138565,
    verifyingContract: '0x0000000000000000000000000000000000000000'
  };
}

// Only one type now — this is the envelope, not the order itself
const TYPES = {
  ExchangeAction: [
    { name: 'payloadHash', type: 'bytes32' },
    { name: 'nonce', type: 'uint64' }
  ]
};

/**
 * Signs any SoDEX trading action (newOrder, cancelOrder, ...) with a
 * registered API key's private key — never the master wallet key.
 *
 * @param actionType  'newOrder' | 'cancelOrder' | 'transferAsset' | ...
 * @param params      action-specific fields, IN SODEX'S EXACT GO STRUCT
 *                    ORDER (see sodex-go-sdk-public) — omit unset optional
 *                    fields entirely, don't send them as null
 * @param apiKeyPrivateKey  the API key's private key (from addAPIKey)
 * @param apiKeyName        the key's registered name → goes in X-API-Key
 * @param market      'spot' | 'futures'
 * @param isMainnet   boolean
 */
export async function getOrderSignatureHeaders(
  actionType, params, apiKeyPrivateKey, apiKeyName, market, isMainnet
) {
  const wallet = new ethers.Wallet(apiKeyPrivateKey);
  const nonce = Date.now(); // must fall within (T-2 days, T+1 day) of block time

  // 1. Build exactly what the server will re-marshal and compare against.
  //    Compact JSON, no whitespace. Key order in params must match the
  //    Go struct field order — JS preserves object-literal insertion order,
  //    so declare the fields in that order when you build params.
  //    DecimalString fields (price, quantity, funds, stopPrice) must be
  //    quoted strings ("0.05"), never numbers.
  const payloadJson = JSON.stringify({ type: actionType, params });

  // 2. Hash it — this replaces putting order fields into the typed struct.
  const payloadHash = ethers.utils.keccak256(ethers.utils.toUtf8Bytes(payloadJson));

  // 3. Sign only {payloadHash, nonce}, not the order.
  const domain = getDomain(market, isMainnet);
  const signature = await wallet._signTypedData(domain, TYPES, { payloadHash, nonce });

  // 4. Typed-signature prefix — this part was already correct.
  const signaturePrefixed = '0x01' + signature.substring(2);

  return {
    'X-API-Key': apiKeyName,        // the key's NAME, not its address
    'X-API-Nonce': nonce.toString(),
    'X-API-Sign': signaturePrefixed,
    'Content-Type': 'application/json'
  };
}
```

### Core Endpoints Used

#### 1. Place Order
Executes a narrative buy or sell trade.

- **Endpoint:** `POST /trade/orders`
- **Headers:** Includes EIP-712 signature headers (generated using `NewOrder` type)
- **Request Body:**
```json
{
  "pair": "BTC-USDC",
  "side": "BUY",
  "orderType": "MARKET",
  "quantity": "0.05000000",
  "price": "0.00"
}
```
- **Example Response:**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "orderId": "ord_8874123490",
    "pair": "BTC-USDC",
    "side": "BUY",
    "quantity": "0.05000000",
    "price": "71842.50",
    "status": "FILLED",
    "filledQuantity": "0.05000000",
    "createdAt": 1783778400000
  }
}
```

---

#### 2. Cancel Order
Used to clear stale stop-loss or limit orders.

- **Endpoint:** `DELETE /trade/orders/{orderId}`
- **Headers:** EIP-712 signature headers (generated using `CancelOrder` type definition)
- **Example Response:**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "orderId": "ord_8874123490",
    "status": "CANCELLED"
  }
}
```

#### 4. Fetch Account State / Balances
Retrieves spot balances and open margin positions.

- **Endpoint:** `GET /accounts/{userAddress}/state`
- **Headers:** EIP-712 signature headers (generated using `GetAccountState` type definition)
- **Example Response:**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "accountId": "acc_0x3F4...9a",
    "walletAddress": "0x3F4a1D2345e67890bCdEf1234567890abcdef123",
    "balances": [
      { "asset": "USDC", "free": "25000.50", "locked": "0.00" },
      { "asset": "BTC", "free": "0.15000000", "locked": "0.00" },
      { "asset": "FET", "free": "420.00000000", "locked": "0.00" }
    ]
  }
}
```

## 🔄 3. End-to-End Integration Flow

The code block below outlines the central handler in `src/engine/trade-engine.js` that brings the two APIs together:

```javascript
import { fetchLatestIndices, fetchNewsFeed } from '../lib/sosovalue.js';
import { executeSodexOrder, fetchAccountBalances } from '../lib/sodex.js';
import { analyzeNarrativeRegime } from './narrative.js';
import { createBreakingStory } from '../lib/openai.js';

export async function checkMarketAndTrade() {
  console.log('🔄 Initiating hourly market evaluation cycle...');

  // 1. Gather intelligence from SoSoValue
  const news = await fetchNewsFeed({ limit: 50 });
  const indexData = await fetchLatestIndices();

  // 2. Classify market regime & narrative temperatures
  const evaluation = await analyzeNarrativeRegime(news, indexData);
  
  if (!evaluation.isShiftDetected || evaluation.confidence < 80) {
    console.log(`✅ Market is stable. Narrative: ${evaluation.dominantNarrative}. Temp: ${evaluation.temperature}°`);
    return;
  }

  console.log(`🚨 ALERT: Narrative shift detected! From [${evaluation.previousNarrative}] to [${evaluation.dominantNarrative}]`);

  // 3. Check account state on SoDEX
  const walletAddress = process.env.USER_WALLET_ADDRESS;
  const balances = await fetchAccountBalances(walletAddress);
  const usdcBalance = parseFloat(balances.find(b => b.asset === 'USDC')?.free || '0');

  if (usdcBalance < 10) {
    console.log('❌ Trading halted: Insufficient USDC balance on SoDEX (< $10).');
    return;
  }

  // 4. Calculate risk-controlled order params
  const maxAlloc = parseFloat(process.env.MAX_ALLOCATION_PER_TRADE || '0.30');
  const tradeSizeUSD = Math.min(usdcBalance * maxAlloc, usdcBalance);
  
  const targetToken = evaluation.recommendedTargetToken; // e.g. "FET"
  const priceTicker = await getSodexTicker(`${targetToken}-USDC`);
  const targetQty = (tradeSizeUSD / parseFloat(priceTicker.price)).toFixed(4);

  // 5. Execute on-chain trade on SoDEX
  console.log(`⚡ Executing trade: Buy ${targetQty} ${targetToken} on SoDEX (approx. $${tradeSizeUSD.toFixed(2)})`);
  
  const orderResult = await executeSodexOrder({
    pair: `${targetToken}-USDC`,
    side: 'BUY',
    orderType: 'MARKET',
    quantity: targetQty.toString(),
    price: '0.00' // Market orders ignore limit price
  });

  if (orderResult.status === 'FILLED') {
    console.log(`✅ Trade filled successfully! Order ID: ${orderResult.orderId}`);

    // 6. Generate and publish breaking narrative wire story
    const storyId = await createBreakingStory({
      shiftData: evaluation,
      tradeData: orderResult,
      publishedAt: new Date().toISOString()
    });

    console.log(`📰 Breaking story published to wire. Story ID: ${storyId}`);
  } else {
    console.log(`❌ Order placement failed or remained pending. Status: ${orderResult.status}`);
  }
}
```
